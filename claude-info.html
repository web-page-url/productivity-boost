What it does

One App to Rule All Skills

Every AI coding assistant stores skills in its own hidden folder with its own format. Productivity Boost unifies them all into a single, beautiful interface.

🔍

Unified Discovery

Scans all agent skill directories on startup — Claude Code, Cursor, Copilot, Windsurf, Gemini CLI, and more — and displays them in one place.

⚡

Enable / Disable

Toggle skills on and off without deleting them. Disabled skills are safely moved to a .disabled/ subfolder and can be re-enabled anytime.

📦

GitHub Install

Paste any GitHub repository URL. The app discovers available skills via GitHub's recursive tree API and lets you select which ones to install.

📋

Copy Between Agents

Found a great Claude Code skill? Copy it to Cursor or Windsurf in one click. The app handles all path resolution automatically.

🛒

Marketplace

Browse community-published skill packs from mcpmarket.com directly within the app. Search, preview, and install in a single flow.

🔐

GitHub Token

Stores a personal access token locally to lift GitHub API rate limits during skill discovery and installation — no server involved.

Data Model

The Skill Interface

Every skill is represented by a unified TypeScript interface, regardless of which agent it belongs to.

// apps/desktop/src/types/index.ts interface Skill { id: string // "{tool}::{dirName}" — globally unique name: string // From SKILL.md frontmatter, falls back to dir name description: string // Short description of what the skill does domain: string // Category (e.g. "pdf-processing", "git") version: string // Semantic version string tags: string[] // e.g. ["pdf", "documents", "text-extraction"] keywords: string[] // Primary + secondary discovery keywords path: string // Absolute filesystem path tool: string // Agent name: "Claude Code", "Cursor", etc. toolPath: string // Base skills directory for the agent enabled: boolean // false if inside .disabled/ subfolder hasTemplates: boolean // Whether templates/ subdir exists templateCount: number // Number of template files found }

SKILL.md — Frontmatter Format

--- name: My Awesome Skill description: Does something really useful domain: productivity version: 1.0.0 tags: [automation, files, output] triggers: keywords: primary: [automate, run] secondary: [execute, script] --- # Rest is markdown — becomes the README tab in the UI
Electron IPC

window.skillsAPI

All communication between React and Node.js goes through a typed API exposed via Electron's contextBridge.

IPC Channel Description
skills:scanAll Scan all agent directories and return all skills grouped by tool
skills:getReadme Read raw SKILL.md / README content for a skill path
skills:listTemplates List template files in a skill's templates/ subfolder
skills:readTemplate Read the content of a specific template file
skills:toggle Move skill folder to/from .disabled/ to enable/disable
skills:delete Permanently delete a skill directory from disk
skills:listAgentPaths Return the full TOOL_PATHS mapping of all known agents
skills:copyToAgent Copy a skill folder to a different agent's skills directory
skills:discoverSkills Query GitHub recursive tree API to find SKILL.md files in a repo
skills:installFromGitHub Download and install selected skills from a GitHub repository
skills:installProgress Event stream emitted during installation (progress messages)
skills:openInExplorer Open skill folder in the OS file explorer (shell.openPath)
skills:openExternal Open a URL in the default browser (shell.openExternal)
skills:searchMarketplace Call mcpmarket.com search API for skill packs
skills:getGithubToken Retrieve stored GitHub PAT from userData config file
skills:setGithubToken Persist a GitHub PAT to userData config file

Install Flow — 3-Stage State Machine

1

Input Stage

User pastes a GitHub repository URL. The modal validates it and triggers skills:discoverSkills.

2

Select Stage

Discovered skills are displayed with checkboxes. User picks which skills to install and which target agent to install into.

3

Installing Stage

Files are downloaded one by one via raw.githubusercontent.com. Progress messages stream back via skills:installProgress events.
